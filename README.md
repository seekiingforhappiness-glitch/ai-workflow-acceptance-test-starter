## Use it in 20 minutes

1. Complete the workflow contract for one outcome and one accountable owner.
2. Replace the placeholders in the acceptance-test CSV with representative examples.
3. Keep the malformed-input, duplicate-event, dependency-failure, and missing-approval tests.
4. Run the QA checklist before calling the workflow ready.

## What it prevents

- vague claims such as “the AI should work well”;
- duplicate side effects after retries;
- production actions without approval;
- hidden scope expansion;
- completion claims without evidence.
